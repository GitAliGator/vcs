The Mapping Window Concept

Similarity Grid

Imagine a similarity grid where reference chunks (Cref) form rows and generated chunks (Cgen) form columns. Each cell (i,j) represents semantic similarity between chunk pairs.

Diagonal → Band

Perfect alignment creates a precise diagonal of high similarity. Natural language variations cause this diagonal to broaden into a "widening band" - this is your Mapping Window.

Permissible Ranges

Mapping Windows define permissible correspondence regions, accommodating text length differences and local density variations where one dense chunk aligns with several less dense chunks.

Precision Windows (MWprec)

Used when matching generated chunks to reference chunks. Shows which reference chunks each generated chunk can align with, defining permissible search ranges.

Recall Windows (MWrec)

Used when matching reference chunks to generated chunks. Shows which generated chunks each reference chunk can align with, ensuring comprehensive coverage assessment.

What's Next: Best Matching Algorithm

Now that mapping windows define permissible correspondence regions, the next step is to find the optimal chunk alignments within these windows. This requires solving two key challenges:

  • • Column-wise matching: Finding best reference chunks for each generated chunk (precision)
  • • Row-wise matching: Finding best generated chunks for each reference chunk (recall)
  • • Avoiding conflicts: Ensuring stable, consistent alignments across both directions
